# HTTP 메서드의 속성
+ 안전(Safe Methods)
+ 멱등(Idempotent Methods)
+ 캐시가능(Cacheable Methods)

# 안전
+ 호출해도 리소스를 변경하지 않음

# 멱등
+ 한 번 호출하든 두 번 호출하든 몇번을 호출하든 결과가 똑같음
+ 멱등 메서드
+ GET : 한 번 조회하든, 두번 조회하든 같은 결과가 조회됨
+ PUT : 결과를 대체하므로 여러번 해도 최종 결과는 같음
+ DELETE : 결과를 삭제하므로 여러번 해도 최종 결과는 같음
+ POST : **멱등이 아님** ex) PRG
+ 활용
+ 자동 복구 메커니즘
+ 서버가 TIMEOUT 등 정상 응답을 못줬을 때, 클라이언트가 같은 요청을 다시 해도 되는가에 대한 판단 근거
+ 외부요인에 의한 리소스 변경은 고려하지 않음

# 캐시가능
+ 응답 결과 리소스를 캐시해서 사용해도 되는가?
+ GET, HEAD, POST, PATCH 캐시 가능
+ 보통은 GET, HEAD 정도만 캐시 사용
+ POST, PATCH는 본문 내용까지 캐시 키로 고려해야 하는데 구현이 쉽지 않음
